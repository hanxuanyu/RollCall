# 课堂随机点名工具 PRD（一次实现可用版）

## 1. 产品目标
提供一个即开即用的课堂点名与积分管理工具，支持网页端与桌面端双形态运行，保证公平性、可控性与课堂展示效果。

本版本一次性实现完整可用能力，不做阶段拆分。

---

## 2. 运行与部署

### 2.1 服务端（Web 模式）
- 启动 Go 二进制
- 浏览器访问（支持局域网）
- 数据与配置：
  - 数据目录：`./data/`
  - 配置文件：`./config/config.yaml`

### 2.2 桌面端（Wails）
- 打包为 `.exe`
- 内嵌前端 UI
- 数据与配置：
  - 数据目录：`./data/`
  - 配置文件：`./config/config.yaml`
- 离线可用

### 2.3 配置文件（config.yaml）
示例：
```
app:
  port: 8080
  mode: web

feature:
  enableScore: true
  enableAnimation: true

random:
  mode: fair
  avoidRepeatWindow: 5
  weightByScore: false
```

### 2.4 配置热更新
- 使用文件监听（fsnotify）
- 修改 config.yaml 后即时生效：
  - 点名模式
  - 权重策略
  - 功能开关
- 无需重启

---

## 3. 功能模块

### 3.1 班级管理

功能：
- 创建 / 编辑 / 删除班级
- 设置默认班级（全局唯一）

字段：
- id
- name
- is_default
- created_at

行为：
- 进入首页自动选中默认班级

---

### 3.2 学生管理

字段：
- id
- class_id
- name（必填）
- gender（可选）
- score（默认0）
- status（正常/禁用）
- created_at

功能：
- 单条增删改查
- 批量导入 / 导出
- 表格查看 + 搜索 + 筛选

#### 导入
支持 CSV / Excel：
- 模式1：仅姓名
- 模式2：姓名 + 性别 + 积分

规则：
- 自动过滤空行
- 重名处理（可配置：允许 / 去重 / 覆盖）

#### 导出
- 导出当前班级完整数据

---

### 3.3 点名系统（核心）

#### 流程
1. 进入首页
2. 自动加载默认班级
3. 选择人数
4. 点击“开始点名”
5. 词云滚动
6. 停止并展示结果

#### 展示（词云）
- 展示全部学生姓名
- 动态字体（可随机或按权重）
- 平滑动画

#### 点名模式

1）纯随机
- 完全随机

2）公平模式（默认）
- 最近被选中概率降低
- 长期均衡

3）积分权重模式（可开关）
- 根据 score 调整概率

#### 防重复
- 单次不重复
- 可配置最近 N 次不重复

#### 结果展示
- 高亮选中人员
- 固定屏幕（防误操作）
- 支持大字号显示

---

### 3.4 积分系统

功能：
- 单人加/减分
- 批量加/减分

规则：
- 支持负分
- 默认分值可配置

#### 积分记录（必须实现）
字段：
- id
- student_id
- delta
- reason（可选）
- created_at

用途：
- 可追溯
- 可扩展排行榜

---

### 3.5 管理界面

功能：
- 班级切换
- 学生表格管理
- 快速编辑
- 批量操作

UI：
- React + shadcn/ui
- 表格 + 弹窗编辑

---

## 4. 数据存储

### 4.1 存储位置
统一规则：
```
./data/
./config/config.yaml
```

### 4.2 存储方案
推荐使用 SQLite：
- 易维护
- 支持结构化查询
- 便于后期扩展

### 4.3 表结构

#### classes
- id
- name
- is_default
- created_at

#### students
- id
- class_id
- name
- gender
- score
- status
- created_at

#### score_logs
- id
- student_id
- delta
- reason
- created_at

#### rollcall_logs（建议）
- id
- class_id
- result（JSON数组）
- created_at

---

## 5. 关键实现点

### 5.1 随机算法
- 基础：rand
- 公平模式：基于“最近命中次数”降低权重
- 权重计算：
  - score
  - 最近命中时间

### 5.2 并发与安全
- 文件写入加锁
- 防止数据损坏

### 5.3 性能
- 支持 1000 人词云流畅运行
- 前端使用虚拟渲染或 canvas

---

## 6. 异常处理

- 无班级：提示创建
- 无学生：提示导入
- 导入失败：返回错误行号
- 数据损坏：自动备份恢复（data/*.bak）

---

## 7. 非功能要求

- 启动时间 < 3 秒
- UI 操作响应 < 200ms
- 动画流畅（60fps 优先）

---

## 8. 验收标准

- 可在 5 分钟内完成部署
- 可完成完整课堂点名流程
- 教师无需学习成本
